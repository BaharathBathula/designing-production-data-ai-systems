# Chapter 6: Cloud Security, Governance, and Compliance

## Overview

Modern data and AI platforms must be secure, trusted, and compliant by design. Governance is not a blocker to innovation; it enables sustainable growth.

## Core Topics

- Identity and access management
- Encryption at rest and in transit
- Zero trust security models
- Data classification
- Metadata and lineage
- Policy automation
- Regulatory compliance frameworks

## Key Principles

- Security by default
- Least privilege access
- Continuous auditability
- Governance with agility

## Key Takeaway

Trustworthy systems are built when security and governance are engineered into every layer.
